Eigenfaces, PCA for Face Recognition¶

The principle component analysis (short PCA) is belonging to the field of unsupervised learning is often used as a method for dimensionality reduction. The idea is very simple and perhaps best explained for a simple 2D situation depicted below.

The left graph shows a collection of data points in the x-y plane. Each data point is connected to \({x_i,y_i}\). We may calculate the standard deviation or variance of the data points of the x-and the y-components to yield the values \(\sigma_x\),\(\sigma_y\). The principle component analysis is now a technique to find uncorrelated variables (new coordinates \(x^\prime\),\(y^\prime\) to successively maximize the variance in the new coordinate system. The right graph in the figure above shows this new coordinate system for the cloud of data points.

Mathematically, this is carrie out by calculating the so called covariance matrix, which contains the variances along the coordinate axes along the diagonal. By finding the eigenvectors and eigenvalues of the covariance matrix we obtain the new coordinate system and the corresponding values.

In the example below, we will carry out a principle component analysis and use it for face recognition. This works, because the same face has some characteristic feature, that is found in each image of this face, while othes faces may show different characteristic features.

A face may be represented by an image \(x_i\) of a certain size of pixels \((h\times w)\) an we can transform each of these images into a column vector which has a length \(h\cdot w\). To obtain a useful dataset, we remove first the mean value by calculating the mean of all images and subtracting it from all images

mean of all images

\[\psi=\frac{1}{m} \sum_{i=1}^{m} x_{i}\]

subtracted the mean

\[\mathrm{a}_{i}=x_{i}-\psi\]

The new images are now stored in \(a_i\) and can be further processed. From all \(m\) images, which we will use for the PCA, we construct a matrix \(A\),

\[A=\left[\begin{array}{lllll} a_{1} & a_{2} & a_{3} & \ldots . & a_{m} \end{array}\right]\]

which has the size \(h\times w \times m\). From this matrix we calculate the covariance matrix

\[{Cov}=AA^{T}\]

which is of size \((h\times w \times m)\) The covariance matrix is now transformed into a new coordinate system with the vector \(\nu_i\). In this system the data has the variances \(\lambda_i\). Since the \(\nu_i\) are the eigenvectors of the covarance matrix, the projection of the covariance matrix on the eigenvectors can be replaced by a multiplication with a number.

\[A^{T} A \nu_{i}=\lambda_{i} \nu_{i}\]

Thus, this also means

\[\begin{split}\begin{array}{r} A^{T} A \nu_{i}=\lambda_{i} \nu_{i} \\ A A^{T} A \nu_{i}=\lambda_{i} A \nu_{i} \\ C^{\prime} u_{i}=\lambda_{i} u_{i} \end{array}\end{split}\]

# Function to plot images in 3 * 4
def plot_gallery(images, titles, h, w, n_row = 3, n_col = 4):
    plt.figure(figsize =(1.8 * n_col, 2.4 * n_row))
    plt.subplots_adjust(bottom = 0, left =.01, right =.99, top =.90, hspace =.35)
    for i in range(n_row * n_col):
        plt.subplot(n_row, n_col, i + 1)
        plt.imshow(images[i].reshape((h, w))+np.min(images[i]), cmap = plt.cm.gray)
        plt.title(titles[i], size = 12)
        plt.xticks(())
        plt.yticks(())

# Generate true labels above the images
def true_title(Y, target_names, i):
    true_name = target_names[Y[i]].rsplit(' ', 1)[-1]
    return 'true label: % s' % (true_name)

true_titles = [true_title(y, target_names, i) for i in range(y.shape[0])]
plot_gallery(X, true_titles, h, w)

def pca(X):

  print('starting PCA...')
  num_data,dim = X.shape
  print(num_data, dim)
  #center data
  mean_X = X.mean(axis=0)
  for i in range(num_data):
      X[i] -= mean_X

  if dim>20:
      print('PCA - compact trick used')
      M = np.dot(X,X.T) #covariance matrix
      e,EV = linalg.eigh(M) #eigenvalues and eigenvectors
      tmp = np.dot(X.T,EV).T #this is the compact trick
      V = tmp[::-1] #reverse since last eigenvectors are the ones we want
      S = np.sqrt(abs(e))[::-1] #reverse since eigenvalues are in increasing order
  else:
      print('PCA - SVD used')
      U,S,V = linalg.svd(X)
      V = V[:num_data] #only makes sense to return the first num_data

  #return the projection matrix, the variance and the mean
  return V,S,mean_X
